The Main Menu

The primary reason I want to create a main menu is so that the player will be able to set the difficulty for the game. This will simply change the number of incorrect guesses allotted to the player. There are three options I want in the menu: Start Game, Settings, and Exit.

So far, this code will only print out a string for the user, creating a main menu for the user to see. My idea for this is that the user will then type in either a 1, 2, or 3, which will then trigger the corresponding menu option. We’re definitely going to need conditionals for this.

For the time being, let’s code for Exit. I would like for Exit to stop the program from running, and in order to do that, I’m going to set a loop for the main menu, and if the loop is broken, then the program would cancel.

I’ve created a variable called Menu with a value of true, and while Menu is true, the main menu loop will continue to run. I have to set Menu to true before the while loop so that the loop will run in the first place. Then, I made another variable within the loop called menuChoice, which will be equal to whatever the user inputs, as represented by the input() function. If the given input is equal to “3” (the data type that input() returns is always a string, so I have to put quotes around the 3), then Menu is set to false, which would then break the while loop, thus ending the program. Let’s set up the conditionals for Settings.

So, I’ve added another conditional stating that if menuChoice is equal to “2”, then print out the “Settings: …” dialogue and request user input again. Difficulty is set to equal the user input, and more conditionals are set. If difficulty is equal to “1”, then the program will set the amount of guesses (amtGuesses) to 9. If difficulty is equal to “2”, then it will set amtGuesses to 6, and if difficulty is equal to “3”, it will set amtGuesses to 3. I added two other conditionals, one to return to the main menu, and one that returns a notification if the given input is not one of the four menu options. If difficulty is set to “4”, then the program will continue, which means the current loop (the while Menu == True loop), is ended and it starts over from the top, effectively putting us back at the main menu.


You may have noticed that I set amtGuesses to equal 6 at the top, outside of the while loop. I did this so that if the user does not go into the settings to change the difficulty, the difficulty is set to normal by default. The reason why amtGuesses is set outside of the loop is that if it is set within the loop, the difficulty will always be set back to normal, even if the user tries to change it to hard or easy. Before I talk about the conditional I created, I want to address if, elif, and else. Converted to English, when using an if-then statement, you are essentially saying “if this is true, then do this”. An else statement attached to the if-then statement is saying “if this is true, then do this, or else, do this”. But what is the point of elif? I had to look this one up out of curiosity. I didn’t see what elif brought to the table that if couldn’t bring.

Fortunately, I live in an age where the internet has answers to just about every possible question. Googling the question brought me to Stack Overflow, and I got my answer. I think the first two responses [Source] help answer my question best, so I’ll try to re-explain it here:

  1. The first distinction is that when you have multiple if statements in a row, the computer will check every if statement to check if it is true. So if I were to run a code like this:

    Now, when I run the program, if I were to type in 15, it would return “This number is less than 21”, as expected. If I typed in 24, it would return “This number is greater than 18”, as expected. But what if I were to type 19? 19 is both less than 21 and greater than 18. With this code, it would return both statements, because both if conditionals return true. Now, let’s change the second if to an elif.

    Now, if I type in 19, it only returns “This number is less than 21”. Why is that? If we refer back to what elif means, “else if”, we can see that elif will only be run assuming if the conditional above it is false. If the above is true, the computer won’t even bother trying the elif conditional.
  2. With that in mind, the second distinction is that elif helps save time and computation. If you have a set of conditionals that all start with if, the computer will check every single conditional, even if you want it to stop checking after one of the conditionals return true. So if we look back at our Start Menu, if the user inputs a “2”, then the computer shouldn’t even bother checking the Exit conditional because the input can’t be both “2” and “3”. If we left everything with if statements, then the computer would first check if the input is “1”, then check if it is “2”, and then check if it is “3”.  That means that even if the user inputs a “1”, despite the fact that the first conditional would be satisfied and there would be no need to check the others, the computer would still check the other conditionals. This results in unnecessary computation. With elif, the moment one of the conditionals returns true, the computer stops checking the remaining conditionals. So with the code we have above, if the user inputs a “1”, the computer checks if the first conditional is satisfied, realizes it is satisfied, and stops checking the rest of them, thus shaving off that extra computation.
  3. The final distinction is that elif allows you to tie in multiple conditionals to one else. If I followed five if statements with one else statement, the else statement would only apply to the fifth if statement. Making Hangman

The first thing we want the game to do is pick a random word from the list dictionary. We can do this by picking a random number between 0 and the number of items in the list minus 1. We do this because the first item in a list has a position of 0, and the nth item in a list has a position of (n-1). So by picking a random number between 0 and (n-1), we essentially pick a random word. We can extract the item at position n in a list by doing list[n].

First, we have to import the random module, which is done on the first line of game_code‘s definition. Then, we set word, which represents the unknown word that the player has to guess, to equal an item of dictionary in the position randomly chosen using the random.randint(0, len(dictionary) -1) function. The whole thing is then set to lowercase letters using .lower(). We do this because a string containing a lowercase letter is different from a string containing the capitalized form of the same letter. I’d prefer to avoid a situation where the player guesses “a”, and the answer is deemed incorrect because the word actually contains “A”, not “a”. Now let’s look back at our goals for this program, excluding what we’ve already accomplished:

  • Have a set of underscores representing how many letters there are in a word.
  • Replace underscores with the correct letters that are guessed.
  • Have a list of already guessed letters and if a letter is guessed that has already been guessed, return a string notifying the player without taking away lives.
  • If a player guesses a number, return a string notifying the player without taking away lives.
  • A player will be allowed to attempt to guess the entire word at any point of the game, but if the length of the guess does not match the length of the answer, return a string notifying the player without taking away lives.
  • The game ends if the player guesses the word, or if the number of guesses hits 0.

The first goal should be our next step. We want there to be as many underscores as there are letters in the word, and we also want spaces between the underscores, otherwise we would have ____ instead of _ _ _ _.

First, we want to provide some sort of output directly telling the player how many letters are in the chosen word. We do this using the print function, which prints the length of the word, len(word), converted into a string using str(). The remaining code creates the aforementioned underscores. A list named underscores is created, and the for loop below will add the underscores by adding a “_” to underscores for each letter in the word (letter is arbitrary here; what the for loop really does is go through a loop for each character in the string word). A separator is also created, which acts as a divider between each item in the list. Since we want a space to separate the underscores, separator is set to equal ” “, a string containing only a space. The underscores are then output to the user, using print. To combine the items in underscores into a string, .join(underscores) is used, and by adding separator in front of that, join will add the given separator between each item being joined together.

Some of you may be wondering why I bothered creating a separator when I could have just added “_ ” (with a space after the underscore) to the list instead of just adding “_”.  This is because in the future, I would like to replace the underscores with correctly guessed letters. Let’s take the word FOE, for example. I guess the letter O. In the string FOE, O is in position 1 (0 is the starting position). In that case, I would like to replace underscores[1] with the letter O so that the underscore in the same position is replaced. However, the user would only input “O”, not “O ” with a space. If I were to simply replace underscores[1] with the user input, “O”, then the following underscore would be too close to the letter. To fix that, I would have to add a space to the user input myself each time, which would not be as efficient. Therefore, I add the separator from the very beginning so I don’t have to deal with that.

Looks like it works! Time to move on to the other goals. We’ll want to do something that will process the guesses the player makes.

I created ListOf_Guesses for later, since I would like to have a list of already guessed letters displayed to the player, as stated in the 6th goal. First, I created a while loop that will continue to process guesses unless the amount of guesses remaining is less than or equal to 0, which would then simply end the game. A couple of print statements are made, simply for the player’s sake. The first one prints out guesses, which represents the number of guesses, and the other prints out ListOf_Guesses, which represents the guesses that have already been made. After this information is printed out for the player, the program then prompts the user for an input. Adding “Make a guess: ” inside the parentheses of input simply prints out that string before asking for the input. Guess is set to equal that input after it has been set to lowercase letters by .lower(). That way, when comparing user input to the word, all letters are lowercase, which makes it much easier to deal with. The next step is to set conditionals based on what the user inputs.

There are five different possible conditionals for a user’s input:

  1. The user inputs a string containing a number.
  2. The user inputs a string that is neither the length of the word or a single character.
  3. The user makes a guess that has already been made.
  4. The user attempts to guess the entire word.
  5. The user guesses a single letter that has not already been guessed.

With this, we’ve effectively translated those five conditionals into code. I also ended up moving print(separator.join(underscores)) to within the loop because I wanted it to print it out after each user input. Let’s review the code:

  1. The first conditional states that if the guess is not an alpha character, then print out a string telling the user to guess a letter. Continue tells the code to start over at the top of the loop without continuing to process the rest of the statements in the loop.
  2. The second conditional states that if the guess is both not the length of the word and is also not one character long, then the user will be notified with another printed statement.
  3. In order to create the third conditional, we have to set up a variable and some extra code. This variable is alreadyGuessed, which by default is set to false. The for loop below it essentially compares guess to each w (word) in ListOf_Guess. If guess is equal to w at any point, alreadyGuessed is set to true, and we break out of the for loop. The third conditional then follows: If alreadyGuessed is true, then inform the user that they have already guessed that letter and continue.
  4. The fourth conditional covers the case where the user attempts to guess the entire word. This can be detected by checking if the length of guess matches the length of word. If true, I am going to assume the user is attempting to guess the whole word. Within that conditional are two more conditionals. If guess is equal to word, then print out a statement informing the user of his victory (the code within the print statement is simply for grammatical purposes. If there is only one guess remaining, then don’t add an “es” to the end of “guess”, otherwise add the “es”).  Break stops the loop from running, which would return the user back to the main menu. If guess is not equal to word, then notify the user, subtract 2 guesses from guess (Attempting to guess the whole word should come with some sort of risk after all) and then continue. This must come after the third conditional because we want to check if the word has already been guessed before processing whether it is correct or not.
  5. Finally, the last conditional, an else statement, simply assumes that if the other conditionals are all false, then the user must have guessed a single character.

As you may have noticed, the else statement simply appends guess to ListOf_Guesses and does not process whether or not the guess is correct or not. I didn’t want to do all the code in one step, so we’ll begin working on that now.

My plan for processing single character guesses was to compare the guess to each letter in the word, and then save the positions where the guess matched the letter. After the guess is appended to ListOf_Guesses, I created an empty list positions, where the positions in the word that contain the guess can be saved.

I needed a way to extract the letters in the word so that I could compare them to the guess, so I decided to use an index. I used word[n] to find the letter in the nth position (0-based index) of the word. I would have to compare guess to the letter in the 0th position, then 1st position, all the way up to the nth position. I used the range function to do this. Range generates numbers starting from 0 and up to n, not including n. This works perfectly here because the last position in a word is equal to the length of the word minus one. The figure below explains a little better:

Using a for loop, I can cycle from 0 to the number representing the last position in the word. Then, within the for loop, I created a conditional stating that if guess is equal to the letter in that position, then append the number representing that position to the list positions.

After the positions have all been saved to the list, the following if conditionals help process that list. If the length of the list is greater than 0, then that means the letter was found in the word at least once. In that case, inform the user of how many times the letter was found in the word.

One of our goals was to replace the blank underscores with letters that were correctly guessed. We can do this now by using a for loop that replaces each of the underscores with a position in positions with the correctly guessed letter. When we return to the top of the while loop, the newly edited underscores will be printed. You may notice that this conditional does not include a break or continue, so there is nothing to interfere with the original while loop. I will clarify the reason for this a little later on.

The else statement refers to the case that positions is empty, or rather the length of the list is not greater than 0, meaning that the letter guessed by the user is not found in the word. In that case, inform the user that the letter is not found in the word, and then subtract 1 from guesses. Then continue.

The final if statement in the picture above is the win condition. If it detects that are no “_” left in underscores, then it can correctly assume that the entire word has been guessed. In that case, the user has won, so print out a string informing the user and then break out of the while loop (no need to continue the loop if the user has already won). I also printed separator.join(underscores) because I want the word to be reprinted if the player wins so that the player can see the whole word written out. Two paragraphs ago, I mentioned how I had chosen to omit continue/break from the conditional. This is because if the user guesses a correct letter, then I want the code to check if there are there any “_” left in underscores after replacing the corresponding underscore with a letter. If I left a continue within that if conditional, then it would not check if the win condition has been satisfied.

Now we just have one more short section of code, the loss condition. The condition is simple: if the player runs out of guesses, then the player loses.

This if condition has to be set outside the while loop because the while loop only runs when guesses is greater than 0. By putting it outside the while loop, then when guesses is less than or equal to 0, the while loop stops running and this if condition runs.
